DNS — Records, Resolution, Caching

2 min read · source: cn/dns_deep.md

The URL journey hand-waved "DNS resolves the name". This page is the unpacked version — record types, who asks whom, and why it's fast.

Record types (the ones interviews use)

Record Maps Note
A / AAAA name → IPv4 / IPv6 the workhorses
CNAME name → another name alias; can't coexist with other records at the same name (why apex domains can't CNAME)
NS zone → authoritative servers the delegation glue
MX domain → mail servers + priority email routing
TXT name → arbitrary text SPF/DKIM, domain-ownership proofs
SOA zone metadata serial + default TTLs

Recursive vs iterative (who does the walking)

Your machine (stub resolver) asks one recursive resolver (router/ISP/1.1.1.1) and just waits. The recursive resolver walks the tree iteratively:

root server → "ask the .com TLD servers" → TLD → "ask ns1.google.com (NS + glue A record)" → authoritative server → the actual A record.

Analogy: you ask a concierge (recursive) for an address; the concierge phones city hall, the district office, then the building manager (iterative). You made one call; the concierge made three.

Caching — why 3 hops don't happen per click

Every record carries a TTL; every layer caches until expiry: browser → OS → recursive resolver. Ninety-something percent of lookups die in a cache. Low TTL = fast failover, more load; high TTL = the reverse — and "why is my DNS change not visible?" = old TTL still counting down somewhere.

Negative caching exists too (NXDOMAIN is cached, per SOA), which is why typos stay broken for minutes.

Worth one sentence each

  • Transport: UDP 53 normally; TCP for zone transfers and responses >512B/truncated; DoT/DoH encrypt the query path (privacy note in URL journey).
  • Anycast: the 13 root "servers" are hundreds of instances sharing IPs — BGP routes you to the nearest (routing in action).
  • DNS-based load balancing: return different A records per query/geo — cheap, TTL-granularity failover.
  • Attacks name-drop: cache poisoning (why source-port randomization + DNSSEC exist), DNS amplification DDoS.

Drill

  1. dig +trace example.com — narrate each response before running it, then run it.
  2. Why can't example.com (apex) be a CNAME to a CDN, and what do providers do instead? (ALIAS/ANAME flattening at the authoritative server.)
  3. You cut TTL from 3600 to 60 the day before a migration — why the day before? (The old 3600 must expire from caches first.)
