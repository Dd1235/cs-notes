What Happens When You Type a URL

4 min read · source: cn/0.md
  • "google.com" enter
  • url? visit : search
  • determine which protocol and which port to connect to
  • http unencrypted on port 80? or https encrypted on port 443?
  • you have not typed in http or https. browsers used to assume that it will be http by default which caused many man in the middle attacks
  • "http strict-transport-security", its a list that browser keeps in cache, the websites that force clients to only use https
  • if in list use https else use http (unsecure)
  • "google.com" is in the list, so its https
  • if it was not in the list, tcp connection through port 80 different from this port 443 connection
  • it could be something else, if you typed "mailto:...", then email app would be opened.
  • for a file server you can't just type "myserver.com", you need to tell the browser something like "sftp://myserver.com"
  • "ssh://" -> ssh goes to port 22
  • "dns" : domain name system
  • domain: google.com, protocol: https, port: 443, ip : ?
  • browser cache? hosts? nope
  • DoH enabled? dns over https, dns is a udp service listening on port 53, so anyone can know which domains you are going to, so dns requests are visible to your isp. What is the differnce between dns over tls and dns over https?
  • if you use DoH, you network provider sees no difference between dns requests and other traffic
  • could be DoT etc. this has a different dedicated port
  • assume its disabled , and its standard dns lookup 53
  • send a udp user datagram, to my default dns provider, which is configured on my router.
  • you can change it to be 1.1.1.1 which is cloudflare dns, or 8.8.8.8 which is by google
  • udp request? create an ip packet
  • in layer 3, aa (my mac) | 3333 (random source port) | 10.0.0.2 (my ip) | dns | 1.1.1.1 | 53 | ?? (mac of my gateway/router)
  • encapsulate into a frame, layer 2 thing, what is the mac address? it is not in my subnet, cannot send it locally, cannot know its mac address
  • dont know where to send it? send to gateway. usually send to router if you don't know
  • my router mac address I know is ff, so send it there, I know its ip is 10.0.0.1
  • ip is used for routing across networks, mac is used for delivery on the current local network segment
  • ARP: address resolution protocol, your machine broadcasts who has this ip? every device hears it and owner replies
  • so at every hop, destination ip is googles, but mac changes at every hop
  • I don't ask what the mac address of 8.8.8.8 is, I ask the mac address of my next hop
  • at the router, I can't send this packet as it is. I use NAT and change the source IP address to the public IP. Add an entry to my nat table that 44.1.2.4->3333 should be 10.0.0.2->3333

  • now lets establish the tcp connection
  • three way handshake, 4.1.2.3 at port 443
  • I dont know the mac address since 4.1.2.3 is not in my subset
  • Send it to my gateway ie my router
  • again private ip -> publich ip, add to nat table
  • 3 way handshake say happened.
  • if my client has a proxy, if https, destination is going to be address of the proxy, not the google
  • assume we establish one tcp connection, we may estabilsh 5 other, how browser does pipelining
  • TLS 1.3, establish a public key and a private key, diffie hellman, send in the publick key and the merged key
  • ALPN application-layer protocol negotiation. extension that I support http2 and what not.
  • if its chrome it could be using quic protocol
  • send back the hello, server hello/ change cipher/ fin
  • generates the symmetric key
  • http/2 uses streams
  • compress into binary format, get/index.html
  • content-type html sent back

The whole journey, in one picture

graph TD
    URL["type google.com + Enter"] --> P{"URL or search?"}
    P --> HSTS{"in HSTS list?"}
    HSTS -- yes --> HTTPS["https, port 443"]
    HSTS -- no --> HTTP["http, port 80 (insecure)"]
    HTTPS --> DNS{"IP cached?<br/>browser / OS / hosts"}
    DNS -- no --> RES["UDP :53 to resolver (1.1.1.1)<br/>→ root → TLD → authoritative"]
    DNS -- yes --> TCP
    RES --> TCP["TCP 3-way handshake<br/>(ARP for gateway MAC, NAT at router)"]
    TCP --> TLS["TLS 1.3 handshake<br/>Diffie-Hellman → symmetric key, ALPN picks h2"]
    TLS --> REQ["HTTP/2 GET /index.html"]
    REQ --> RENDER["response → parse → render"]
