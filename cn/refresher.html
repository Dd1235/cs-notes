TCP vs UDP in one table

TCP UDP
Connection 3-way handshake, stateful none, stateless
Delivery ordered, reliable, retransmits fire and forget
Unit byte stream (no message boundaries) datagram (boundaries preserved)
Overhead 20+ byte header, ACKs, state 8 byte header
Head-of-line blocking yes no
Use when correctness matters: web, DB, mail latency matters: DNS, video, games, QUIC

Handshake & teardown

  • SYN → SYN-ACK → ACK: exchanges initial sequence numbers; both sides allocate state.
  • Teardown: FIN → ACK → FIN → ACK (4 steps, both directions close independently).
  • TIME_WAIT: the side that closes first waits ~2×MSL so stray old segments die and the final ACK can be retransmitted. It is normal, not a bug.

Flow control vs congestion control

Flow control protects the receiver (rwnd, advertised in every ACK).
Congestion control protects the network (cwnd, inferred from loss/RTT).

  • Sender may have in flight: min(rwnd, cwnd).
  • Slow start: cwnd doubles every RTT until threshold or loss.
  • Congestion avoidance: additive increase, multiplicative decrease.
  • Fast retransmit: 3 duplicate ACKs → resend without waiting for timeout.

select vs poll vs epoll

select poll epoll
FD limit 1024 (FD_SETSIZE) unlimited unlimited
Cost per call O(n) copy + O(n) scan O(n) copy + scan O(1) — ready list
Kernel state none (rebuilt each call) none persistent interest list
Portability everywhere everywhere Linux (kqueue on BSD/mac)
  • The problem they solve: "10,000 sockets, which are readable, without 10,000 threads?"
  • Level-triggered: notified while data remains. Edge-triggered: notified once on arrival — must read until EAGAIN.

Zero-copy in one breath

Normal send: disk → page cache → user buffer → socket buffer → NIC (2 wasteful copies through userspace). sendfile() keeps data in the kernel; with DMA scatter-gather the CPU copies ~0 times. This is why nginx and Kafka are fast.

URL → page, the checklist

  1. Parse scheme (HSTS list may force https)
  2. DNS: browser cache → OS cache → resolver → root → TLD → authoritative
  3. TCP handshake (port 443), then TLS handshake
  4. HTTP request → response → parse → render
  5. Sockets: socket() → bind() → listen() → accept() server-side

Drill questions

  1. Why is TCP a byte stream and what bug does that cause in naive code? (message framing is your job)
  2. Why does TIME_WAIT exist and on which side?
  3. rwnd = 0 — what happens? (sender pauses; window probes)
  4. Why is epoll O(1) when poll is O(n)?
  5. Why does binding to 0.0.0.0 in dev deserve a code-review comment?
  6. Difference between congestion loss signals: timeout vs 3 dup ACKs?
  7. Why is UDP used for DNS but TCP for zone transfers?
