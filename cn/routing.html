Routing — Longest Prefix, OSPF, BGP

2 min read · source: cn/routing.md

A router's job in one sentence: match the destination IP against a table of prefixes, pick the longest match, forward out that interface. The rest is how the table gets filled.

Longest-prefix match

Destination 10.1.2.7, table has 10.0.0.0/8 → eth0 and 10.1.2.0/24 → eth1 → the /24 wins (more specific). 0.0.0.0/0 is the default route — matches everything, loses to anything. (This lookup is what routing hardware optimizes with tries/TCAM — a nice bridge if asked "where do tries show up in real life".)

Two worlds of routing protocols

Inside one organization (IGP) vs between organizations (EGP) — different problems, different algorithms:

OSPF — link-state, inside

Every router floods "here are my links and their costs" to all others; each builds the full map and runs Dijkstra locally for shortest paths. Fast convergence, needs trust and shared metrics — perfect inside one company. (RIP is the older distance-vector cousin: "my neighbor said 3 hops, so I say 4" — slow convergence, count-to-infinity; know it as contrast.)

Analogy: OSPF = everyone gets the complete city map and computes their own routes. Distance-vector = navigating by rumors from adjacent intersections.

BGP — path-vector, between

The internet is ~80k autonomous systems (ASes) that don't trust each other and route by policy (money), not distance. BGP advertisements carry the full AS path ("reach 8.8.8.0/24 via [15169]"); loop prevention = "my own AS number is already in the path, discard". Route choice is policy-ranked (local preference, path length, …) — a business decision, not a shortest path.

Why it matters beyond trivia: BGP misconfigurations take countries offline (route leaks/hijacks — the classic YouTube-via-Pakistan story), and "why is BGP slow to converge" is a real Arista conversation.

Drill

  1. Table: /0 → A, 10/8 → B, 10.1/16 → C. Where do 10.1.5.5, 10.2.0.1, 8.8.8.8 go? (C, B, A.)
  2. Why can't OSPF run the internet? (No universal metric, no policy, flooding doesn't scale across 80k untrusting ASes.)
  3. Traceroute shows your packet leaving via a different ISP than expected — which BGP knob likely explains it? (Local preference.)
