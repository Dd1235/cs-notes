IP Addressing & Subnetting

2 min read · source: cn/ip_subnetting.md

Subnetting is one skill: split an IP into network bits and host bits, then count. Do the /26 example below twice and you own it.

The idea

An IPv4 address is 32 bits. A prefix (/24, CIDR notation) says how many leading bits name the network; the rest name hosts inside it. The old class A/B/C system was fixed prefixes (/8, /16, /24) — CIDR made the split arbitrary and killed the classes; mention classes only as history.

Analogy: an address is street + house number. The prefix length decides where street ends and house begins — longer prefix, more streets, smaller streets.

The mechanics (worked /26)

192.168.1.0/26 → 26 network bits, 6 host bits → 2⁶ = 64 addresses per subnet.

  • Block size in the last octet: 256 − 192(mask .192) = 64 → subnets start at .0, .64, .128, .192.
  • Each subnet loses 2: network address (all host bits 0) and broadcast (all 1s) → 62 usable hosts.
  • Which subnet is 192.168.1.100/26? 100 falls in [64,128) → network .64, broadcast .127, usable .65–.126.

That's the entire exam: block size = 256 − mask octet; round down to find your block.

Prefix Mask Addresses Usable
/24 255.255.255.0 256 254
/26 255.255.255.192 64 62
/27 255.255.255.224 32 30
/30 255.255.255.252 4 2 (router links)
/31, /32 — 2, 1 point-to-point, single host

Special ranges (recognition speed matters)

  • Private (RFC 1918): 10/8, 172.16/12, 192.168/16 — what NAT hides (URL journey).
  • Loopback 127/8; link-local 169.254/16 (= "DHCP failed" in the wild).
  • IPv6 in one breath: 128 bits, no NAT needed, no broadcast (multicast instead), ::1 loopback, SLAAC auto-config.

Why routers care

Forwarding = longest-prefix match: a route to 10.1.2.0/24 beats 10.0.0.0/8 for 10.1.2.7. Subnetting is what makes the table hierarchical instead of one row per host — see routing.

Drill

  1. 172.16.35.200/20 — network, broadcast, usable range? (Block 16 in third octet: 172.16.32.0 – 172.16.47.255.)
  2. Smallest subnet fitting 500 hosts? ( /23 → 510 usable.)
  3. Why does a /31 work for router-to-router links despite "minus 2"? (RFC 3021 — no broadcast needed on point-to-point.)
