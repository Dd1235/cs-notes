Exceptions & Error Handling

3 min read · source: oops_cpp/self_notes/18.md

  • 70% of effort goes into testing and debugging and retesting

  • pushing to a full stack, ctrl+c, socket event, logical errors like pop from an empty stack, resource errors like memory read/write, run time errors like artimentic overflow or underflow, out of range, division by error etc.

  • exceptions are C++ means of separating error report from error Handling

  • current state saved in a pre-defined location

asynchronous exceptions

  • unexpected, eg, interrupt

synchronous

  • planned, happen exactly at that point in program, like when popping from an empty stack
  • implemented as a throw
  1. error incidence
  • synchronous (s/w) local error
  • async (h/w) interrupt or s/w interrupt
  1. create object and raise exception
  • an exception object can be of any complete type
  • an int to a full blowk c++ class object
  1. detect exception
  • polling software tests
  • notification control adjustment
  1. handle exception
  • ignore: hope someone else handles it, do not catch
  • act: but allow others to handle it afterwards. catch, handle, rethrow
  • or own the exception. take system to consistent state. catch and handle
  1. recover from exception
  • continue excution: if handled inside the program
  • abort execution: if handled outside the program

RAII removes its nead.

#include <stdio.h>
#include <stdlib.h>
#include <errno.h>

int process_file(const char *path) {
    FILE *file = NULL;
    char *buffer = NULL;
    int result = -1;   // assume failure by default

    file = fopen(path, "r");
    if (file == NULL) {
        goto error;    // jump to common error handler
    }

    buffer = malloc(1024);
    if (buffer == NULL) {
        goto error;    // file must still be closed
    }

    if (fread(buffer, 1, 1024, file) == 0) {
        goto error;    // both buffer and file must be cleaned
    }

    // Success path
    result = 0;
    goto cleanup;

error:
    // C has no exception object like C++.
    // We manually set/return an error code.
    errno = EINVAL;
    result = -1;

cleanup:
    // Shared cleanup section.
    // Safe because free(NULL) does nothing,
    // and we check file before closing.
    free(buffer);

    if (file != NULL) {
        fclose(file);
    }

    return result;
}

Meaning: many checks can fail from different places, but all failures go to one labeled error-return block. One neat trapdoor instead of ten tiny trapdoors.
