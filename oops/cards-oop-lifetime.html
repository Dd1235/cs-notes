OOP, Lifetime & Memory Flashcards

9 min read · source: oops_cpp/flashcards/02_oop_lifetime.md

O001

Tags: oop

Prompt: Define class, object, state, behavior, and identity.

Answer: A class defines a type. An object is a runtime instance. State is stored data. Behavior is member functions/operations. Identity means two objects can have equal state but still be distinct objects.

O002

Tags: oop, design

Prompt: Encapsulation vs information hiding?

Answer: Encapsulation groups data with operations. Information hiding prevents users from depending on representation, preserving invariants behind a minimal interface.

O003

Tags: oop, access

Prompt: Default access for class and struct?

Answer: class defaults to private. struct defaults to public. Conventionally, use struct for passive aggregates and class for invariant-owning abstractions.

O004

Tags: oop, access

Prompt: What can access public, protected, and private members?

Answer: public: everyone. protected: the class, derived classes, and friends. private: the class and friends.

O005

Tags: oop, this

Prompt: What is the hidden this pointer, and why do static member functions lack it?

Answer: Every non-static member function receives a hidden pointer to the invoking object. Static member functions belong to the class, not an object, so there is no this.

O006

Tags: constructors

Prompt: Why prefer member initializer lists?

Answer: They directly initialize members, are required for references and const data members, avoid default-construction then assignment, and express dependencies more clearly.

O007

Tags: constructors, edge-case

Prompt: In what order are data members initialized: declaration order or initializer-list order?

Answer: Declaration order in the class, regardless of initializer-list order. If one member depends on another, declare them in dependency order.

O008

Tags: constructors, explicit

Prompt: What does explicit prevent on a single-argument constructor?

Answer: It prevents implicit user-defined conversions such as converting an int into Meter accidentally during function calls or operator expressions.

O009

Tags: lifetime, destructors

Prompt: What does a destructor do, and when is it called?

Answer: It releases resources/finishes object lifetime. It runs at scope exit for automatic objects, during delete, for members/bases during enclosing object destruction, and during stack unwinding.

O010

Tags: raii

Prompt: What is RAII?

Answer: Resource Acquisition Is Initialization: acquire a resource in an object's constructor and release it in the destructor, so cleanup is automatic even on exceptions.

O011

Tags: lifetime

Prompt: Destruction order for local objects, members, and bases?

Answer: Locals are destroyed in reverse construction order. After a destructor body runs, members are destroyed in reverse declaration order, then base classes in reverse construction order.

O012

Tags: destructors, exceptions

Prompt: Why should destructors generally not throw?

Answer: If a destructor throws during stack unwinding from another exception, the program may call std::terminate. Destructors should preserve cleanup guarantees.

O013

Tags: copy-control

Prompt: State Rule of 3, Rule of 5, and Rule of 0.

Answer: Rule of 3: if a class manually manages a resource and defines destructor, copy constructor, or copy assignment, it likely needs all three. Rule of 5 adds move constructor and move assignment. Rule of 0 prefers RAII members so compiler-generated special members are correct.

O014

Tags: copy-control

Prompt: Copy constructor vs copy assignment?

Answer: Copy constructor creates a new object from an existing one. Copy assignment replaces the state of an already-existing object and should return *this.

O015

Tags: copy-control, edge-case

Prompt: Why is Point(Point other) not a valid copy-constructor style?

Answer: Passing by value would require copying other, which calls the copy constructor again, causing infinite recursion.

O016

Tags: copy-control

Prompt: Which are copy constructors: X(const X&), X(X&), X(const volatile X&), X(X*)?

Answer: Reference forms to X with possible cv-qualification are copy constructors. Pointer forms such as X(X*) are not copy constructors.

O017

Tags: copy-control, edge-case

Prompt: What is the shallow-copy trap for a class owning char*?

Answer: The compiler-generated copy copies the pointer value, not the pointed buffer. Two objects then share one allocation, causing double free, aliasing, or dangling pointers.

O018

Tags: copy-control, edge-case

Prompt: Why can naive copy assignment break on s = s?

Answer: If it first frees s.str_ and then copies from s.str_, it reads from freed memory. Handle self-assignment or use copy-and-swap.

O019

Tags: copy-control, exception-safety

Prompt: What does copy-and-swap assignment look like conceptually?

Answer: Take the right-hand side by value, swap its resources with *this, and return *this. The temporary cleans old state on exit and self-assignment is naturally safe.

O020

Tags: move-semantics

Prompt: What does std::move actually do?

Answer: It casts an expression to an rvalue reference. It does not move by itself; a move constructor or move assignment must be selected.

O021

Tags: move-semantics

Prompt: Why should move operations often be noexcept?

Answer: Containers such as std::vector can use noexcept moves during reallocation while preserving exception guarantees. If move may throw, they may fall back to copying.

O022

Tags: move-semantics

Prompt: In a move constructor for a raw pointer owner, why set the source pointer to nullptr?

Answer: Ownership has been transferred. Nulling the source prevents its destructor from deleting the same allocation.

O023

Tags: const

Prompt: What does double length() const mean for this?

Answer: Inside the function, this behaves like a pointer to const object, so non-mutable data members cannot be modified and only const member functions can be called.

O024

Tags: const, mutable

Prompt: When is mutable appropriate?

Answer: For data that may change without changing the logical value of the object, such as caches, memoization flags, or synchronization fields.

O025

Tags: const, mutable, edge-case

Prompt: Can mutable apply to static, reference, or const data members?

Answer: No. mutable applies to non-static data members affected by object constness; static members do not belong to an object.

O026

Tags: static

Prompt: What are static data members and static member functions?

Answer: Static data members are one class-wide object, not per-instance. Static member functions have no this and can directly access only static members.

O027

Tags: singleton

Prompt: Give the modern local-static Singleton skeleton.

Answer:

class Config {
public:
    static Config& instance() {
        static Config c;
        return c;
    }
    Config(const Config&) = delete;
    Config& operator=(const Config&) = delete;
private:
    Config() = default;
};

O028

Tags: singleton, design

Prompt: Why can Singleton be harmful?

Answer: It introduces hidden global state, complicates testing, creates lifetime-order risks, and may become a dumping ground. Dependency injection is often cleaner.

O029

Tags: friend

Prompt: What does friend grant, and is the friend a member?

Answer: friend grants access to private/protected members. The friend function/class is not a member, so access labels around the friend declaration do not matter.

O030

Tags: friend, edge-case

Prompt: Is friendship inherited, symmetric, or transitive?

Answer: No. If A friends B, that does not friend derived classes of B, does not friend A back, and does not friend B's friends.

O031

Tags: friend, design

Prompt: When is friendship reasonable?

Answer: For tightly coupled operations that are logically part of the abstraction but should be non-members, such as symmetric arithmetic operators, stream insertion, or iterator/list internals.

O032

Tags: operators

Prompt: What can and cannot be changed by operator overloading?

Answer: You can define function behavior for existing operators involving user-defined types. You cannot create new operators or change arity, precedence, associativity, or built-in operator behavior for only built-in operands.

O033

Tags: operators

Prompt: Which operators must generally be members?

Answer: operator=, operator[], operator(), and operator-> must be non-static member functions.

O034

Tags: operators

Prompt: Why are stream operators usually non-member functions?

Answer: The left operand is std::ostream or std::istream, which you cannot modify. A non-member operator<< can take the stream first and return it by reference for chaining.

O035

Tags: operators

Prompt: Why should operator<< return std::ostream&?

Answer: Returning the stream by reference enables chaining: cout << a << b.

O036

Tags: operators

Prompt: Why does 2 + complex often require a non-member/friend overload?

Answer: In a member operator, the left operand must be the invoking object. 2 is not a Complex, so a non-member overload can accept (double, const Complex&).

O037

Tags: operators

Prompt: Prefix vs postfix increment overload signatures and return style?

Answer: Prefix: T& operator++() and usually returns *this by reference. Postfix: T operator++(int) with dummy int, returns the old value by value.

O038

Tags: operators, edge-case

Prompt: What happens if &&, ||, or comma are overloaded?

Answer: They lose some built-in special properties such as short-circuit evaluation/order behavior expected from built-in logical operators, so overload with extreme care.

O039

Tags: namespace

Prompt: What is a namespace for?

Answer: A lexical scope for grouping names and avoiding collisions. Namespaces can contain functions, classes, variables, templates, and nested namespaces.

O040

Tags: namespace

Prompt: Are namespaces open? Are classes namespaces?

Answer: Namespaces are open and can be reopened for more declarations. A class also defines a scope for its members, but every namespace is not a class and cannot be instantiated.

O041

Tags: namespace, linkage

Prompt: What does an unnamed namespace do?

Answer: It gives internal linkage to its contents, making them visible only within that translation unit. It is broader and cleaner than C-style file-scope static.

O042

Tags: dynamic-memory, new

Prompt: Distinguish the new expression/new operator, operator new, and placement new.

Answer: A new expression allocates storage and constructs an object. operator new(size_t) is the allocation function, like malloc for raw storage. Placement new new (buffer) T(args) constructs an object at a supplied address and does not allocate.

O043

Tags: dynamic-memory, delete

Prompt: Distinguish delete expression and operator delete.

Answer: A delete expression calls the destructor and then deallocates storage. operator delete(void*) is only the deallocation function for raw storage.

O044

Tags: dynamic-memory, placement-new, edge-case

Prompt: How do you clean up an object created with placement new?

Answer: Manually call the destructor, e.g. p->~T();. Do not call delete p unless the storage itself came from the matching ordinary new path and you are deliberately managing that full protocol.

O045

Tags: dynamic-memory, placement-new

Prompt: Why do containers allocate raw storage before constructing elements?

Answer: Capacity may exceed size. Constructing every reserved slot would waste work/resources. Containers allocate raw storage, then placement-construct only live elements.

O046

Tags: dynamic-memory, arrays

Prompt: Why must new[] match delete[]?

Answer: Array new has separate allocation/destruction semantics and must destroy every element. Using scalar delete on an array is undefined behavior.

O047

Tags: dynamic-memory, edge-case

Prompt: Is delete nullptr; safe?

Answer: Yes. Deleting a null pointer has no effect.

O048

Tags: class, namespace

Prompt: Why can two classes both have a member called data without conflict?

Answer: Each class defines its own class scope, so names are qualified as A::data and B::data.
