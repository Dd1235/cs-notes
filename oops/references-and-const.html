References & const

3 min read · source: oops_cpp/self_notes/1.md

  • NULL, its just 0

  • we can't have int& i because reference must be initialized

  • can't have int& i = 5, no address to refer to as 5 is a constant. const int& i = 5 is ok because we can refer to the constant value 5

  • int& i = j + k is not ok because j+k is a temporary value and we can't refer to it. const int& i = j + k is ok because we can refer to the temporary value as a constant reference

  • j+k produces a temporary object (rvalue / prvalue), its ephemeral. non-const reference emas you have a stable mutable object. const int& works because binding a const reference to a temporary extends the lifetime

  • c uses return by value , we can make in cpp return by reference so we get an alias of the return variable. const int& b = foo() is ok because we can bind a const reference to the temporary returned by foo() and extend its lifetime. int& b = foo() is not ok because we can't bind a non-const reference to a temporary.

  • Value Parameter: input : call by value

  • reference parameter: in-out : call by reference

  • const reference parameter: input : call by reference

  • return value: output: return by value, return by reference

  • we still have call by value for backward compatibility, and because integer and character may be in register and not in any memory location, so call by value is simple register copy with one cycle

  • we may get deferencing null pointer error but not deferencing null reference because that situation won't arise

  • pointer can point to null.

  • pointers allow user to operate on the address, diff pointers, increment etc. references don't. all operations are interpreted for the referent.

  • array of pointers can be defined, but array of references not allowed

  • a reference must bind immediately. But arrays require default construction possibility. Element assignment rearrangement semantics.

  • a reference is an alias in the languge semantic model.

  • like having a constant pointer without the syntactic overload of a pointer. Your referent cannot be changed.

void foo(const int x)

this doesn't affect caller but still local copy of x can't be modified.

int identity(int a = 10){
    return (a);
}

can call identity without parameter with default value. Specify values from left, so the right parameters will get defaulted valued.

Default parameters should be supplied only in a header file and not in the definition of a function.

Default arguments are handled at call site, not inside the function. So caller must know the default value.

Default arguments cannot be redefined.

  • default parameter case is a special case of function overloading.

  • semantic dichotomy in c

  • array variable in the context of a call, compiler implements a call by reference

  • if you pass a struct or union it will be call by value
