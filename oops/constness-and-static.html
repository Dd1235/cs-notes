Const-ness & Static Members

3 min read · source: oops_cpp/self_notes/9.md

if its a const function its like mytype const _ const this is passed instead of mytype _ const this.

compiler checks if that function really doesn't modify data member. const objects can only call const member functions.

a constant data member cannot be changed even in non constant object. eg id of a student.

also

#include <bits/stdc++.h>
using namespace std;

class Base {
public:
const int x = 5;
};

class Child : public Base {
public:
int x = 10;
};

int main() {

    Child c;

    cout << c.x << "\n";         // 10
    cout << c.Base::x << "\n";   // 5

}

this is name hiding, not overriding.
functions can be overridden. variables cannot.

Base* b = new Child;
cout << b->x;

this resolved using Base::x because member variable are resolved using the statyc type of hte pointer unlike virtual functions that use runtime dispatch.

There may be a constant object where there are some data members may be changed. mutable is provided to model logical constness against the default bit-wise constness.

mutable is applicable to only data member not variables.
reference, static, and const data members cannot be declared as mutable.

even const member functions can modify mutable data members.

class A {
public:
    mutable int cache_hits = 0;

    int value = 10;

    int get() const {
        cache_hits++;
        return value;
    }
};

mutable applies to object constness while static members do not belong to an object at all. so mutable static int x gives error.

static data members

  • must be defined in global scope. non static data members do not need separate definitions, instantiated with the object.
  • static data members are initalized during program startup.
  • MyClass::x exists as one global variable for the whole program.

Singleton class

  • can only have one instance
  • eg POTUS or Director of IIITB

How to implement a singleton class?

  • constructor is private, member function that can invoke the constructor. now catch 22, constructor is private, can't make object and have member function. So static member function so that instance of object is not needed.

class Printer {
    bool blackAndWhite_, bothSided_;
    Printer(bool bw = false, bool bs = false) : blackAndWhite_(bw), bothSided_(bs) {}

    static Printer *myPrinter_;

public:

    ~Printer() {
        cout << "Destroying printer\n";
    }
    static const Printer& printer(bool bw = false, bool bs = false) {
        if (myPrinter_ == nullptr) {
            myPrinter_ = new Printer(bw, bs);
        }
        return *myPrinter_;
    }
}

Caller receives read only reference.

This makes sure illegal construction is prevent at compile time and memory allocation and construction process isn't illegally started. Copy constructor problem also taken care of?

But this singleton is not thread safe either.
Byt static Printer instance; is thread safe, compiler internally generates synchronization.

= delete -> function is forbidden to use.

My singleton also broken by copy problem. Printer::myPrinter_ doesn't change, but multiple instances can be created.

Proper singleton

class Printer {

private:

    Printer() {}

    static Printer* myPrinter_;

public:

    Printer(const Printer&) = delete;
    Printer& operator=(const Printer&) = delete;
};
