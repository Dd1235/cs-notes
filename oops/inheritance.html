Inheritance & Overriding

1 min read · source: oops_cpp/self_notes/12.md

  • Code reuse

  • x isa y, y is a generalization of x, x is a specialization of y

  • manager isa employee => single inheritance

  • two wheeler isa vehicle, threewheeler isa vehicle => hybrid inheritence

  • red rose isa rose isa flower -> multilevel inheritance

  • public inheritance means "is-a". everything that applies to a base class but also apply to derived class.

  • a ctor of derived must first call ctor of base class.

  • Derived class can access protected member of Base class

  • no other class or global function can access protected member of Base class


class B {
    protected: int data_;
    public:

    // becomes implicitly inline like member functions defined inside class

    friend ostream& operator<<(ostream& os, const B& b) {
        os << b.data_ <<endl;
        return os;
    }
}

class D : public B {
    int info_;

};
D d(1, 2);
cout << d; // 1

will need another friend function in D to print info
