Multiple & Private Inheritance

2 min read · source: oops_cpp/self_notes/13.md

  • multiple inheritance: one class being specialized from two.

  • private inheritance means nothing during software design, only during software implementation

  • class Derived : private Base;

  • if using private or protected inhertance, it is NOT a "specialization / generalization " case!

  • has an "is implemented in terms of" semantics

  • in public inheritance, derived class object can be converted intoa. base class object by compiler

  • in private inheritance, derived class object cannot be converted into a base class object by compiler.

class Person {..}
class Student : private Person {..}
void eat(const Person& p) {..}
int main() {
    Student s;
    eat(s); // error: cannot convert from Student to Person
}

Protected inheritance is similar to private inheritance, except that the public members of the base class become protected members of the derived class, and the protected members of the base class remain protected in the derived class. This means that the derived class can access the public and protected members of the base class, but they are not accessible to code outside the derived class.

eluding concept without much practical use.

this is composition or private inheritance?

if private inheritance, everything becomes private. if external to car wants to use engines start method

Private inheritance is inferior to composition, makes sense when a derived class needs access to protected base class members or needs to redefine inherited virtual function.

  • casting is performed when a value of one type is used in place of some other type.

  • double result = d / i; // i is cast to double and used, mixed mode arithmetic, implicit conversion

  • implit conversion will give warning

  • if you explicitly cast, there won't be warning

  • with two unrelated classes it won't work.

  • can explicty cast pointer of one type to pointer of another type, but it is not recommended.

class A {int i};
class B {double d};
A a;
B b;

A *p = &a;
B *q = &b;

a = b; // error, right hand operand of type main::B
a = (A)b; // error, cannot convert
p = q; // error
p = (A*)&b; // forced - okay

Upcasting implictly works.
Downcast gives error.

if void* is used, it can point to any type. But we lose the type. Now upcasting also won't work.
