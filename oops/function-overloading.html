Function Overloading

2 min read · source: oops_cpp/self_notes/4.md

Function Overloads:

  • same name, different parameters

  • very convinient, imagine we had types row_vector, col_vector, matrix, etc.

  • Function Overloading -> static polymorphism

  • this binding, which call to what function definition is decided at the compile time.

  • In C++ we can also have operator overloading, which is a special case of function overloading.

  • To resolve overloaded functions with one parameter, identify a set of candidate functions, from them viable set, then select best viable.

  • Exact Match -> promption -> standard conversion -> user defined conversion

  • if overload resolution fails, we get compile time error.

  • function overloading can use default parameters.

int area(int a, int b) ..
double area(double a, double b) ..
double z = 2.5;
int x = 2;
area(z,x); // calls the second because exact match at first parameter, promotion at second

default parameters are not part of the function signature.

lvalue => value of the variable, identifiable location in memory, int x = 10, x is an lvalue
rvalue -> address of the variable, temporary value, no persistent identity, cannot assign them

x++ => returns old value temporarly, rvalue
++x -> return object itself, lvalue

in cpp no way to distinguish, it depends on the context.

int& b = 5; // error, cannot bind non-const lvalue reference to an rvalue
const int& a = 5; // ok, can bind const lvalue reference to an rvalue, extend temporary lifetime
int&& c = 5; // ok, can bind rvalue reference to an rvalue, c itself is a lvalue
int&& d = x; // error, cannot bind rvalue reference to an lvalue

&& is part of the TYPE
we may use int&& for move semantics, perfect forwarding, etc.

Binary operators are infix, function is always prefix.

forna operator there is an associated function, and that can be overloaded.

in c++ we have a new keyword operator to define operator overloads.

a+b => operator+(a,b)
a=b => operator=(a,b)
c=a+b => operator=(c, operator+(a,b))

enum E { C0 = 0, C1 = 1, C2 = 2 };

E operator+(const E* a, const E* b) {
    return static_cast<E>((static_cast<int>(*a) + static_cast<int>(*b)) % 3);
}
  • even << >> can be opeoverloaded.
  • ++ can be prefix or postfix,
type& operator++(type& s) => prefix
type operator++(type& s, int) => postfix, the int is a dummy parameter

::, .*, sizeof, ?: cannot be overloaded.

name mangling by compiler

foo(int)        -> _Z3fooi
foo(double)     -> _Z3food
foo(int,int)    -> _Z3fooii

usually stored in teh text segment.

g++ test.cpp -c
nm test.o

can see the mangled names.
