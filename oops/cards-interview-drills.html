Interview & Code-Reading Drills

12 min read · source: oops_cpp/flashcards/07_interview_drills.md

I001

Tags: trap, lifetime

Prompt: What is wrong?

int& bad() {
    int x = 42;
    return x;
}

Answer: It returns a reference to a local variable whose lifetime ends when the function returns. The reference dangles; using it is undefined behavior.

I002

Tags: trap, virtual-destructor

Prompt: What is wrong?

struct Base { ~Base() {} };
struct Derived : Base { std::vector<int> v; };
Base* p = new Derived;
delete p;

Answer: Base lacks a virtual destructor. Deleting a derived object through a base pointer is undefined behavior.

I003

Tags: trap, macros

Prompt: What is printed or why is this bad?

#define SQR(x) ((x) * (x))
int i = 2;
int y = SQR(i++);

Answer: It expands to ((i++) * (i++)); i is incremented twice and behavior/result is not what a function-like square should mean. Prefer an inline/template function.

I004

Tags: trap, slicing

Prompt: What happens?

struct B { virtual void f(); };
struct D : B { void f() override; };
void g(B b) { b.f(); }
D d;
g(d);

Answer: d is sliced into a B object when passed by value. The call is on the sliced base object, so it cannot dispatch to D::f.

I005

Tags: trap, move

Prompt: Does std::move(a) move data by itself?

Answer: No. It casts a to an rvalue reference. A move happens only if a move constructor/assignment or overload is selected.

I006

Tags: trap, static-type

Prompt: What does this show?

struct Base { int x = 5; virtual void f(); };
struct Child : Base { int x = 10; void f() override; };
Base* b = new Child;
std::cout << b->x;
b->f();

Answer: b->x uses Base::x because data members bind by static type. b->f() dispatches to Child::f because f is virtual.

I007

Tags: trap, casts

Prompt: Why is this wrong when calling base logic from an override?

void SpecialWindow::onResize() {
    static_cast<Window>(*this).onResize();
}

Answer: It casts by value, creating a sliced Window temporary, then calls on that temporary. Use Window::onResize();.

I008

Tags: trap, casts

Prompt: Multiple choice: Which cast should be used for a runtime-checked downcast in a polymorphic hierarchy?

Options: A. static_cast B. dynamic_cast C. const_cast D. reinterpret_cast

Answer: B. dynamic_cast.

I009

Tags: trap, dynamic-cast

Prompt: What is the failure behavior?

Base* b = new Dog;
Cat* c = dynamic_cast<Cat*>(b);
Cat& r = dynamic_cast<Cat&>(*b);

Answer: Pointer cast failure returns nullptr. Reference cast failure throws std::bad_cast.

I010

Tags: trap, const-cast

Prompt: Is this safe?

const int x = 5;
int* p = const_cast<int*>(&x);
*p = 6;

Answer: No. The original object is truly const, so modifying it through p is undefined behavior.

I011

Tags: trap, typeid

Prompt: With a polymorphic base pointer Base* p = new Derived;, what is the conceptual difference between typeid(p) and typeid(*p)?

Answer: typeid(p) reports the static pointer type Base*. typeid(*p) reports the dynamic object type Derived because the expression is a polymorphic glvalue.

I012

Tags: trap, pure-virtual

Prompt: Can this be useful?

struct Shape {
    virtual void draw() = 0;
};
void Shape::draw() { /* common setup */ }

Answer: Yes. Shape remains abstract, but derived overrides can call Shape::draw() for shared setup.

I013

Tags: trap, copy

Prompt: Why is this dangerous?

struct S {
    char* p;
    ~S() { delete[] p; }
};
S a;
S b = a;

Answer: The compiler-generated copy constructor shallow-copies the pointer. Both objects own the same allocation and will double-delete.

I014

Tags: trap, copy-assignment

Prompt: What property should copy assignment support for a = b = c?

Answer: It should return *this by reference so assignment chaining works naturally.

I015

Tags: trap, self-assignment

Prompt: What breaks in naive assignment?

String& operator=(const String& s) {
    delete[] data;
    data = clone(s.data);
    return *this;
}

Answer: On s = s, it deletes its own data and then tries to clone from freed memory. Check self-assignment or use copy-and-swap.

I016

Tags: trap, initializer-order

Prompt: What is the bug?

struct S {
    size_t len_;
    char* str_;
    S(const char* s) : str_(strdup(s)), len_(strlen(str_)) {}
};

Answer: Members initialize in declaration order, so len_ initializes before str_; strlen(str_) reads an uninitialized pointer. Declare str_ before len_ or compute from s.

I017

Tags: trap, placement-new

Prompt: What is wrong?

alignas(T) unsigned char buf[sizeof(T)];
T* p = new (buf) T();
delete p;

Answer: Placement new used stack-provided storage. delete p incorrectly tries to deallocate it. Call p->~T(); only.

I018

Tags: trap, new

Prompt: Fill in: new T(args) roughly means __ then __.

Answer: Call allocation function operator new(sizeof(T)), then construct T in that storage, conceptually with placement new.

I019

Tags: trap, delete

Prompt: Fill in: delete p roughly means __ then __.

Answer: Call p->~T() if non-null, then call the matching deallocation function operator delete(p).

I020

Tags: trap, dynamic-memory

Prompt: Which are mismatched and undefined: malloc/free, new/delete, new[]/delete, new/free?

Answer: malloc/free and new/delete are matched. new[]/delete and new/free are mismatched and undefined.

I021

Tags: trap, operator-overloading

Prompt: Why can a + 2 work with a member operator but 2 + a may not?

Answer: Member operators require the left operand to be the invoking class object. 2 is not that class. Use a non-member/friend overload for symmetry.

I022

Tags: trap, operator-overloading

Prompt: Why can't operator<< for Complex be a Complex member if you want cout << c?

Answer: The left operand is cout, so the operator would need to be a member of ostream, which you cannot add. Use a non-member overload.

I023

Tags: trap, operator-overloading

Prompt: Which postfix increment signature is correct?

Options: A. T& operator++() B. T operator++(int) C. T operator++(T) D. T& operator++(int&)

Answer: B. The dummy int distinguishes postfix.

I024

Tags: trap, namespaces

Prompt: What does ::data mean?

Answer: It qualifies data in the global namespace.

I025

Tags: trap, namespaces

Prompt: What is the difference between using namespace ns; and using ns::x;?

Answer: using namespace ns; makes all names in ns available for lookup. using ns::x; brings only x into the current scope.

I026

Tags: trap, friends

Prompt: If A is a friend of B, can B access A's private members?

Answer: Not unless A also declares B as a friend. Friendship is not symmetric.

I027

Tags: trap, friends

Prompt: If A friends B, are derived classes of B automatically friends?

Answer: No. Friendship is not inherited.

I028

Tags: trap, exceptions

Prompt: What is printed?

struct B {};
struct D : B {};
try { throw D{}; }
catch (B) { cout << "B"; }
catch (D) { cout << "D"; }

Answer: B. The base catch appears first and catches the derived exception by value; the D catch is unreachable.

I029

Tags: trap, exceptions

Prompt: Why is catch (std::exception e) weaker than catch (const std::exception& e)?

Answer: Catching by value copies and slices derived exception objects. Catching by const reference preserves polymorphic behavior and avoids copying.

I030

Tags: trap, templates

Prompt: Why does a macro Max(a++, b++) have a side-effect problem but a function template does not?

Answer: A macro may substitute each argument multiple times. A function template evaluates each argument once before the call.

I031

Tags: trap, templates

Prompt: How do you write a template that knows the size of a raw array argument?

Answer:

template<class T, size_t N>
T max_array(const T (&arr)[N]);

Taking the array by reference prevents decay.

I032

Tags: trap, templates

Prompt: Why might typename T::name* p; be required?

Answer: Because T::name is a dependent nested type. Without typename, the parser can treat it as a value expression.

I033

Tags: trap, stl

Prompt: Why does remove_if(v.begin(), v.end(), pred); alone not shrink v?

Answer: It only partitions/moves elements and returns the new logical end. You must call erase to remove the trailing elements physically.

I034

Tags: trap, stl

Prompt: Why might list be slower than vector despite O(1) insert/erase with iterator?

Answer: Poor cache locality and allocation overhead can dominate. vector's contiguous storage is often much faster in practice.

I035

Tags: trap, functors

Prompt: Why can sort(v.begin(), v.end(), IsGreaterThan{5}); use state while a raw function pointer cannot?

Answer: The functor object stores limit = 5 as member state. A plain function pointer is only an address and has no per-call state.

I036

Tags: trap, uml

Prompt: In a class diagram, which relation means the part cannot normally outlive the whole?

Answer: Composition.

I037

Tags: trap, uml

Prompt: Which UML diagram answers "what messages happen over time between objects"?

Answer: Sequence diagram.

I038

Tags: trap, uml

Prompt: Which UML diagram answers "which operations are valid in each lifecycle state"?

Answer: State machine diagram, or protocol state machine for interface method-call validity.

I039

Tags: trap, patterns

Prompt: You need undo/redo for editor actions. Which pattern and why?

Answer: Command, because each action can be represented as an object with enough state to execute and undo.

I040

Tags: trap, patterns

Prompt: You need light/dark/platform-specific Button, Checkbox, and Menu families. Which pattern?

Answer: Abstract Factory, because it creates families of compatible related products.

I041

Tags: trap, patterns

Prompt: You have an AST and frequently add new passes like print, optimize, and type-check, but rarely add node kinds. Which pattern?

Answer: Visitor.

I042

Tags: trap, patterns

Prompt: You have a global logger. Should it always be Singleton?

Answer: Not always. Singleton may be acceptable for a truly unique resource, but dependency injection usually improves testability and makes dependencies explicit.

I043

Tags: trap, testing

Prompt: A test executes every statement but asserts almost nothing. What is the risk?

Answer: High statement coverage but weak fault detection. Coverage alone does not prove meaningful behavior is checked.

I044

Tags: trap, testing

Prompt: For input range 1..100, why test 0, 1, 100, 101, and a middle value?

Answer: Boundary value analysis: defects cluster at edges and just outside valid ranges.

I045

Tags: trap, se

Prompt: Scope increases, time and cost are fixed. What usually happens?

Answer: Quality suffers unless scope is renegotiated, resources change, or delivery is split.

I046

Tags: combined, design

Prompt: A leave-management system has many leave types and approval rules. What design avoids a giant switch and stays testable?

Answer: Model leave rules as LeavePolicy strategy/policy objects, route approvals with a separate ApprovalWorkflow, and use a factory to select policies if creation varies.

I047

Tags: combined, cplusplus-design

Prompt: You write a base interface for polymorphic deletion and copy should be forbidden. What special members do you declare?

Answer: virtual ~Base() = default;, delete copy constructor/assignment if copying through the interface is invalid, and consider protected/defaulted constructors as appropriate.

I048

Tags: combined, exception-safety

Prompt: For a custom string assignment, how do copy-and-swap and RAII provide the strong guarantee?

Answer: Copy the new value first. If copying throws, the original object is unchanged. Then swap resources; the temporary destroys the old resource.

I049

Tags: combined, casts-design

Prompt: An interview asks "which cast should I use for downcasting?" What is the best complete answer?

Answer: Prefer redesigning the interface to avoid downcasting. If a checked downcast is genuinely needed in a polymorphic hierarchy, use dynamic_cast. static_cast downcast is unchecked and risky.

I050

Tags: combined, lifetime

Prompt: Why does vector<T> need placement new internally?

Answer: It separates capacity from size: allocate raw storage for many Ts, construct only live elements as needed, destroy live elements manually, then deallocate raw storage.

I051

Tags: combined, const

Prompt: A const MyString has both char& operator[](size_t) and const char& operator[](size_t) const. Which is chosen?

Answer: The const overload, because a const object can call only const member functions. It returns const char&, preventing modification through the result.

I052

Tags: combined, overloads

Prompt: What is the natural return type for operator= and why?

Answer: T&, returning *this, because assignment changes an existing object and supports chaining like a = b = c.

I053

Tags: trap, inheritance, static-type

Prompt: Why does this not compile?

class Person {};
class Student : public Person {
public:
    void cry();
};

Person* p = new Student;
p->cry();

Answer: The static type of p is Person*, and Person has no cry member. C++ member lookup is based on static type first. Virtual dispatch only chooses among functions declared in the base interface.

I054

Tags: trap, slicing

Prompt: Why does this not let you call Student::cry()?

Person* p = new Student;
Person q = *p;
q.cry();

Answer: q is a separate Person object created by slicing the Student. It has no derived part and no cry member.

I055

Tags: trap, vector, unsigned

Prompt: What is wrong with this reverse loop?

for (int i = v.size() - 1; i >= 0; --i) {
    use(v[i]);
}

Answer: v.size() is size_t (unsigned). If the vector is empty, v.size() - 1 wraps to a huge value before conversion. Signed/unsigned comparisons can also surprise. Use a safer idiom such as for (size_t i = v.size(); i-- > 0;).

I056

Tags: trap, vector, unsigned

Prompt: Why can if (i < v.size()) be risky when i is signed?

Answer: Usual arithmetic conversions may convert i to unsigned. A negative i can become a very large value, producing surprising comparisons.

I057

Tags: trap, custom-vector

Prompt: In a hand-written MyVector<T>, why track both _size and _capacity?

Answer: _size is the number of constructed/live elements. _capacity is allocated storage available before reallocation. This allows amortized O(1) push_back.

I058

Tags: trap, custom-vector

Prompt: What are the missing production concerns in a simple MyVector implemented with new T[new_capacity] and copy assignment?

Answer: It default-constructs all capacity slots, needs correct copy/move constructors and assignment, must handle exceptions during resize, should destroy only live elements, and should use allocators/raw storage plus placement new for full STL-like behavior.

I059

Tags: trap, custom-vector

Prompt: In custom vector operator[], what is wrong with if (index >= size)?

Answer: size is likely a member function, not the current element count. The condition should compare with _size or size().
