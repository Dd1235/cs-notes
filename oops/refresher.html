Rapid Refresher

3 min read · source: website/content/oops_refresher.md

A last-pass revision sheet distilled from the OOP/C++ notes and flashcards. The lecture notes carry the detail; this is the skeleton to hang it on.

The four pillars, said like an engineer

  • Encapsulation — state + behavior together; invariants guarded behind an interface.
  • Abstraction — expose what, hide how (information hiding).
  • Inheritance — is-a specialization; reuse of interface, not just code.
  • Polymorphism — one call site, many behaviors: static (overloading/templates, compile time) vs dynamic (virtual functions, runtime).

References & value categories

  • A reference must be initialized; int& r = 5; is illegal — no object to alias.
  • const int& r = j + k; is legal: binding a const reference to a temporary extends its lifetime.
  • int&& binds to rvalues — the hook that enables move semantics.

Object lifetime cheatsheet

  • Construction order: bases → members (in declaration order, not initializer-list order) → constructor body. Destruction is exact reverse.
  • Copy constructor runs on initialization; copy assignment on assignment to an existing object.
  • Rule of 3/5/0: manage a raw resource → define destructor + copy ctor + copy assign (+ move pair); better, own nothing raw and define none.
  • delete vs delete[] must match new vs new[].

Virtual dispatch, mechanically

  • Each polymorphic class has a vtable; each object carries a vptr set during construction.
  • obj->f() through a base pointer: follow vptr → vtable slot → call. One extra indirection.
  • Static type is what the compiler sees; dynamic type is what the object is.
  • Destructor of a base class must be virtual if you delete through a base pointer.
  • Virtual calls inside constructors don't dispatch to the derived class — the derived part doesn't exist yet.

The four casts

Cast Use Checked
static_cast implicit-style conversions, up/down without check compile time
dynamic_cast safe downcast in polymorphic hierarchy runtime (nullptr / throws)
const_cast add/remove const — only legal if object isn't originally const no
reinterpret_cast bit reinterpretation; last resort no

Exceptions

  • Throw by value, catch by (const) reference.
  • Stack unwinding destroys locals — this is why RAII works: resources release themselves on any exit path.
  • Never let an exception escape a destructor.

Templates & functors

  • Function/class templates = compile-time polymorphism; each instantiation is a separate type.
  • Functor = object with operator() — inlinable state-carrying callable; lambdas are compiler-written functors.
  • Template metaprogramming: the compiler as interpreter — types computed during compilation.

Design patterns most asked

  • Singleton — one instance, global access (thread-safe: Meyers' static local).
  • Factory / Abstract Factory — construction behind an interface.
  • Observer — subject notifies subscribers (pub/sub).
  • Strategy — swap algorithm at runtime via an interface.
  • Adapter / Decorator — reshape vs enrich an interface.
  • Principle behind most: program to an interface; prefer composition over inheritance (SOLID).

Drill questions

  1. Why does a constructor have no return type — argue via this.
  2. f() is non-virtual, g() is virtual; base pointer to derived object calls each — what runs and why?
  3. Why is copying a vector of raw pointers with the default copy ctor a bug? Name the fix (deep copy / rule of 3 / smart pointers).
  4. Why can't static_cast be used to safely downcast? When would you accept it anyway?
  5. What is object slicing and how do references/pointers avoid it?
  6. Sketch the memory of an object with one virtual function and one int.
