Classes & Objects

1 min read · source: oops_cpp/self_notes/6.md

Object -> data members -> state
Object -> member functions -> behavior

Identity -> this pointer holds the address of the current object, this pointer is implicitly passed to all non-static member functions of a class

struct only aggregates, class aggregates + helps to do more for building udt

C structs cannot have member functions.
Simulate methods using function pointers

#include <stdio.h>

struct Point {
    int x, y;

    void (*print)(struct Point*);
};

// now I cannot define another printPoint function. It's not in a namespace.
void printPoint(struct Point* p) {
    printf("%d %d\n", p->x, p->y);
}

int main() {
    struct Point p = {1, 2, printPoint};

    p.print(&p);
}

A lot of oop in c is buit this way, Linux Kernel objects, GUI toolkits, embedded systems etc.

class => namespace

this pointer: X * const this, this is a constant pointer to a non constant object of type X
only accessible in the class

class Point {
    int x, y;
    void foo() {} // Point * const this
    void bar() const {}; // type of this: const Point * const
};
const Point* p = new Point(2,3);
p->foo(); // not allowed, this function may mutatte the object

Point q; // allocates memory for q on the stack

Private part of a class -> forms its implementation
Public -> interface to others

state should be changes only through methods
behaviour should be accesssible by methods
