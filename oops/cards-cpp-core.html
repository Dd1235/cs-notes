C++ Core Flashcards

6 min read · source: oops_cpp/flashcards/01_cpp_core.md

C001

Tags: cpp, build

Prompt: List the four main stages of the C++ build pipeline.

Answer: Preprocessing expands includes/macros/conditionals; compilation turns each translation unit into object code; assembly emits machine instructions; linking combines object files/libraries and resolves symbols.

C002

Tags: cpp, build, headers

Prompt: What is a translation unit?

Answer: One source file after preprocessing, including all headers textually included into it. The compiler compiles translation units independently.

C003

Tags: cpp, headers

Prompt: What belongs in a header vs a .cpp file?

Answer: Headers usually contain declarations, type definitions, inline definitions, templates, and include guards. .cpp files contain non-template function/object definitions that should exist once.

C004

Tags: cpp, odr

Prompt: What is the One Definition Rule?

Answer: A program should have exactly one definition of each non-inline entity across the linked program. Identical inline and template definitions may appear in multiple translation units.

C005

Tags: cpp, linkage

Prompt: Match each linkage construct: extern int x;, file-scope static, anonymous namespace, extern "C".

Answer: extern int x; declares a variable defined elsewhere. File-scope static gives internal linkage. Anonymous namespace is the modern internal-linkage mechanism for names in a translation unit. extern "C" disables C++ name mangling for C interoperability.

C006

Tags: cpp, headers

Prompt: Why should non-template function definitions generally not be put in headers?

Answer: Including the header in multiple translation units creates multiple definitions at link time unless the function is inline and all definitions are identical.

C007

Tags: cpp, templates, headers

Prompt: Why are template definitions usually placed in headers?

Answer: The compiler needs the full template body at the use site to instantiate code for the concrete template arguments.

C008

Tags: cpp, mangling

Prompt: Why does C++ use name mangling?

Answer: C++ supports overloading, so linker symbols encode type/signature information to distinguish functions such as f(int) and f(double).

C009

Tags: cpp, memory

Prompt: Name typical process memory regions and what they store.

Answer: Text/code stores instructions; read-only data stores string literals/constants; data stores initialized globals/statics; BSS stores zero-initialized globals/statics; heap stores dynamic allocations; stack stores call frames and automatic locals.

C010

Tags: cpp, lifetime

Prompt: What is the lifetime of a local automatic variable, a static local variable, and a heap object?

Answer: Automatic local: from declaration execution to scope exit. Static local: initialized once and lives until program termination. Heap object: from construction via new/allocator until destroyed/freed by the owner.

C011

Tags: cpp, arrays

Prompt: What does "array decay" mean?

Answer: In many expressions, an array converts to a pointer to its first element. After decay, size information is lost, so sizeof(parameter_array) inside a function usually measures pointer size, not array length.

C012

Tags: cpp, containers

Prompt: Prefer which standard type for each need: fixed-size sequence, dynamic contiguous sequence, text, raw byte buffer ownership.

Answer: std::array<T, N>, std::vector<T>, std::string, and std::vector<std::byte> or a smart pointer.

C013

Tags: cpp, algorithms

Prompt: What are the key preconditions/complexities for sort, find, and binary_search?

Answer: std::sort needs random-access iterators and is usually O(n log n). std::find is linear. std::binary_search needs a sorted range and is logarithmic comparisons.

C014

Tags: cpp, const

Prompt: Fill in: const means __; constexpr means __; volatile means __.

Answer: const means not modifiable through that name. constexpr means evaluable at compile time when inputs allow. volatile means the value may change outside normal program flow; it is not a thread-synchronization primitive.

C015

Tags: cpp, const, pointers

Prompt: Decode const int* p, int* const p, and const int* const p.

Answer: const int* p: pointer can move, pointee cannot be modified through p. int* const p: pointer cannot move, pointee can be modified. const int* const p: neither pointer nor pointee can be modified through p.

C016

Tags: cpp, macros

Prompt: Why is #define SQUARE(x) ((x) * (x)) unsafe for SQUARE(i++)?

Answer: The macro textually substitutes i++ twice, so i is incremented twice and the expression has surprising side effects.

C017

Tags: cpp, inline

Prompt: In modern C++, what does inline primarily mean?

Answer: It permits multiple identical definitions across translation units. It does not force the compiler to inline machine code.

C018

Tags: cpp, references

Prompt: Pointer vs reference: list three practical differences.

Answer: A pointer can be null, can be rebound, and requires explicit dereference. A reference is normally a non-null alias, cannot be rebound, and is cleaner for required parameters.

C019

Tags: cpp, references, lifetime

Prompt: When is return-by-reference safe, and what is the classic bad example?

Answer: Safe only when the referred object outlives the call, such as returning v[0] from a vector passed by reference. Bad: returning a reference to a local variable.

C020

Tags: cpp, parameters

Prompt: When would you pass a parameter as T, T&, const T&, or T*?

Answer: T for cheap copies or ownership-independent values; T& for required mutable argument; const T& for required read-only large objects; T* for optional/maybe-null or pointer semantics.

C021

Tags: cpp, functions

Prompt: What is the rule for default arguments?

Answer: Defaults are filled by the caller at compile time, must appear from right to left, and should usually be written in the declaration rather than repeated in definitions.

C022

Tags: cpp, overloading

Prompt: Order common overload-resolution preference levels.

Answer: Exact match, promotion, standard conversion, user-defined conversion, variadic fallback.

C023

Tags: cpp, dynamic-memory

Prompt: Contrast new/delete with malloc/free.

Answer: new allocates and constructs; delete destroys and deallocates. malloc only allocates raw bytes and returns void*; free only releases bytes and does not call destructors.

C024

Tags: cpp, dynamic-memory

Prompt: Which pairs must match: malloc, new, new[], raw operator new?

Answer: malloc with free; new with delete; new[] with delete[]; raw operator new with operator delete after manually constructing/destructing as needed.

C025

Tags: cpp, undefined-behavior

Prompt: Name eight common sources of undefined behavior.

Answer: Out-of-bounds access, use-after-free, double delete, null dereference, returning reference to local, signed integer overflow, data race, and reading uninitialized storage.

C026

Tags: cpp, stdlib

Prompt: Why is std::vector usually preferred over a raw dynamic array?

Answer: It owns memory via RAII, tracks size/capacity, handles construction/destruction, supports bounds-checked access via at, and integrates with algorithms through iterators.

C027

Tags: cpp, namespaces

Prompt: Why should using namespace std; not be placed in headers?

Answer: It pollutes every translation unit that includes the header, creates name conflicts, and silently changes overload/name lookup for users.

C028

Tags: cpp, scope-lifetime

Prompt: Scope vs lifetime: what is the difference?

Answer: Scope is where a name is visible in source code. Lifetime is when the object/storage exists at runtime. They often overlap but are not the same.
