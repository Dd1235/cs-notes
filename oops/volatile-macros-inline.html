volatile, Macros & inline

2 min read · source: oops_cpp/self_notes/0.md

volatile

Think memory-mapped I/O vs I/O-mapped I/O: a device register lives at a memory address, and the hardware may change the value of that variable behind the compiler's back. volatile tells the compiler: don't optimize reads/writes away, don't cache it in a register — actually touch memory every time.

volatile uint32_t* status = reinterpret_cast<uint32_t*>(0x4000'0000);
while (*status == 0) { }   // without volatile the compiler hoists the load → infinite loop

Note for interviews: volatile is not a threading primitive in C++ — it gives no atomicity and no ordering. Threads need std::atomic.

Debug vs production builds

We also have debug builds (-O0 -g, assertions on) and production builds (-O2/-O3, NDEBUG). Behavior differences between them are usually optimizer-exposed bugs — which is exactly why debuggability of your constructs matters:

Don't use #define for constants

  • Use const int x = ...; (or constexpr) or an inline function instead.
  • A macro is very bad for debugging: the preprocessor does textual substitution before the compiler runs, so the debugger has no symbol, error messages point at expanded garbage, and there's no type checking or scoping.
  • #define is the preprocessor mechanism; inline is a language keyword — an inline function is real code the compiler understands (typed, scoped, steppable) that can still be expanded at the call site with zero overhead. That's the whole trade: macro speed without macro pain.
#define SQR(x) x * x          // SQR(a + b) → a + b * a + b   — silently wrong
inline int sqr(int x) { return x * x; }   // typed, scoped, debuggable

const and pointers — the line trick

Draw a line through the * — whatever const sits next to is what's constant:

const int* p = &x;   // non-constant pointer to a const int   (can't do *p = 5)
int* const q = &x;   // constant pointer to a non-const int   (can't do q = &y)
const int* const r = &x;   // both frozen

Read right-to-left: q is a const pointer to int; p is a pointer to int that is const.
