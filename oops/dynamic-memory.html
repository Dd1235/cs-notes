Dynamic Memory Management

4 min read · source: oops_cpp/self_notes/5.md
  • malloc() and free() in C
  • don't know the size of the memory needed, at runtime can acquire more memory from the os, once done we can free them
  • linkedlist may not know how many nodes will be there etc.

C++

  • new and delete
  • Array new[] and delete[]
  • Placement new()
  • Restrictions
  • Overloading new and delete

Malloc is part of stdlib.h, needs to be provided size of memory in bytes, cstdlib in C++
int *p = (int *)malloc(sizeof(int))
free(p)

pointer to allocated memory returned as void* -> needs to be cast as int*
malloc() returns NULL if it fails to allocate memory, so we should check for that before using the pointer.
malloc cannot be initialized.

calloc() and realloc() are available in both languages.

int* p = new int(4)
delete p
Operator new for allocation on heap. No size specification needed, only type. Allocated memory returned as int*, no casting required. Can be initiazlied.
Operator delete for deallocation from heap.

It's a core language feature, no header needed.

int *p = (int *)operator new(sizeof(int));
operator delete(p)

these are the operator functions and hence can be overloaded. operator new behaves much like malloc and needs the szie.

operator+
operator[]
operator()
operator int
operator<<

these are all special declaractor forms.

There is a major different between operator new and operator new()

Operator new function behaves like malloc. Operator new also constructs the poerator.
Operator delete first calls the destructor of the class and then deallocates the memory.


int *a = (int*)malloc(sizeof(int)*3);
int *b = new int[3]; // different version of new operator for arrays
free(a);
delete [] b;
// buf is on the stack frame
unsigned char buf[sizeof(int)*2]; // buffer on stack
// placement new in buffer buf
int *pInt = new (buf) int (3); // placement new
int *qInt = new (buf+sizeof(int)) int (5);

new : 1) allocate memory, 2) call constructor
delete: 1) call destructor, 2) deallocate memory

placement new - skip 1, you provide the memory yourself.

int* p = new int(5);

// becomes roughly

void* mem = operator new(sizeof(int));
new (mem) int(5);

pass in the pointer to the memory where you want to construct the object.

If you have done placement new, cannot be deleted with delete. If you have allocated on stack, cannot use delete.

malloc() -> free()
operator new -> operator delete
operator new[] -> operator delete[]
operator new -> placement new -> no delete

passing NULL to delete is secure
The new operator allocates exact amount of memory from heap.

new, new[], delete, delete[] have separate semantics.

This allows delayed constructions.

vector<T> internally allocates raw memory first. We don't need 1000 constructors running. So raw memory block first, only few objects constructed.

T arr[100] would construct all 1000 immediately.
Disaster for performance or resources.

void* operator new(size_t n){
    cout << "Overloaded new" << endl;
    void *ptr;
    ptr = malloc(n);
    return ptr;
}
void operator delete(void* ptr){
    cout << "Overloaded delete" << endl;
    free(ptr);
}

new is much better than just the operator new(), it takes in the type instead of the size, and returns pointer corresponding to that type.
But can't overload this functionality, we don't have higher order function in C++. We can't pass in type as a parameter to a function. So we can only overload the operator new() and not the new operator itself. We can write a wrapper function around the operator new() to achieve similar functionality, but it won't be as elegant as overloading the new operator itself.
