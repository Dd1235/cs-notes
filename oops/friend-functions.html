Friend Functions & Friend Classes

3 min read · source: oops_cpp/self_notes/10.md

  • Lot of debate if this should be there in a purely object oriented language.

  • inside my class friend void fun(const A& a); => add the signature inside the class, define it outside void func(const A& a) { cout << a.x; }

  • non-member functions but friend to a class. Able to access private member.

  • breaks encapsulation.

  • friend can be global function, or a member function of a class, or a function template.

  • friendship ignores access sections. doesn't matter is signature is in public, private or protected section. It can access all members of the class. Because friend is not a class member so label doesn't matter.

Friendship is not inherited, not symmetric, not transitive.

May need forward declaraction if there is circular dependency. Say a function needs to be friend of both class A and class B, need forward declaraction.

class Node;

class List {
    Node* head;
    Node * tail;

public:
    List(Node* h = 0) : head(h), tail(h) {};
    void display();
    void append(Node* p);
};

class Node {
    int info;
    Node* nxt;
public:
    Node(int i): info(i), nxt(0){};
    friend void List::display();
    friend void List::append(Node* p);
};

void List::display() {
    Node* p = head;
    while (p) {
        cout << p->info << " ";
        p = p->nxt;
    }
}

void List::append(Node* p) {
    if (head == 0) {
        head = tail = p;
    } else {
        tail->nxt = p;
        tail = p;
    }
}

One class needs to make friendship with a lot of member functions, might as well have a friend class.

A friend class is not within that namespace and not bound by the access specifier.

A friend class can be a class or a class template.

With an iterator

class Node; class List;
class Iterator {
    Node* node;
    List* list;

public;
    Iterator() : node(0), list(0) {};
    void begin(List *);
    bool end();
    void next();
    int data();
};

class List {
..
friend class Iterator;
};

class Node {
    ..
    friend class List;
    frined class Iterator;
};

void Iterator::begin(List *l){
    list = l;
    node = l->head;
}
bool Iterator::end() {
    return node == 0;
}
void Iterator::next() {
    if (node) node = node->nxt;
}
int Iterator::data() {
    if (node) return node->info;
}

friendship is neither commutative nor transitive.

If we made them public, external users can also access that are not friends.
We could have a nested iterator class, nestled classes often get privileged access.

std::vector::iterator is a nested class, it has access to private members of vector.

A language like Java has package-private. All classes in the same package can access each other's private members. So it is like a friend class, but it is not explicitly declared. It is implicit.

C++ values:

  • zero cost abstractions
  • memory layout control
  • avoiding unnecessary getters/setters
  • tight systems level cooperation

a = b => a.operator=(b)
++a => a.operator++()
a++ => a.operator++(int)

For a member operator function, invoking object is implicity passed as the left operand, and the right operand is explicily passed as the argument.
