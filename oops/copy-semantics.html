Copy Constructor & Copy Assignment

2 min read · source: oops_cpp/self_notes/8.md

initialization is not based on initializer list, but based on original class data member order

class Point {
    int x, int y;
    public:
    Point(int y, int x) : y(y), x(x) {}
}

first x is initialized then y.

class String{
    size_t len_;
    char * str_;// container
public:
    String(char *s) : str_(strdup(s)), len_(strlen(str_))
}

crashes, len will be initialized first, as constructor is invoked, str_ is garbage pointer, no proper memory allocation.
Initializer list, if initialization of data members is dependent, then we write them in correct dag order.

ctor = constructor, dtor = destructor

For Point p (1, 2); invokes Constructor Point::Point(int,int);

For Point p2 = p1; invokes Copy Constructor Point::Point(const Point&);

Point p1(1,2), p2(p1),p3=p1; // constructor, copy, copy

call-by-value needs copy. needs copy constructor.
return-by-value also needs copy constructor.

Hence its needed for function call.

Signature of copy constructor can be

Point(const Point& other); // pass by reference, avoid infinite recursion
Point(Point& other); // non-const, can be used for move semantics, but not for copy constructor
Point(volatile const Point& other); // volatile, not common, but can be used for copy constructor
Point(volatile Point& other);

These are not copy constructors

Point(Point* other);
Point(const Point* other);

Why must it be call by reference? Infinite loop
Point(Point other), for call by value we need to make a copy, hence copy constructor would be called and so on.

Compiler provded copy constructor performs bit-copy. Takes the bit layout and lays it out in the object being copied to.

That is fine for say Point.

It will copy the pointer but not the pointed object.

Copy Assignment:

Copy assignment: object already exists
Assigment should support chaining so it has to return the reference to the current object. I can't return a new object.

class String{
    char* str_;

public:
// copy assignment operator
    String& operator=(const String& s){
        free(str_); // release existing memory, without the free, resource will leak
        str_ = strdup(s.str_); // perform deep copy
        return *this; // need to return this object for chaining
    }
};


int main(){
    String s1 = "Hello";
    String s2 = "World";
    s2 = s1; // copy assignment
    s1 = s1; // what happens? free s1.str_, then strdup on freed up object, so its garbage, so this will crash
}

A member function of a class can access private members of any object of the same class, not just its own object.

this is allowed, private to the class, not to the object.

class Point {
    int x, y;

public:

    Point(int a, int b) : x(a), y(b) {}

    Point& operator=(const Point& other) {
        x = other.x;
        y = other.y;
        return *this;
    }
};
