  • eg. Stack<char>
  • class template can be done with a forward declaration
template<class T> class Stack;
Stack<char> s; // is an error
Stack<char> *ps; // is okay, doesn't need to know impl
void ReverseString(Stack<char>& s, char *str); // is okay

template<class T> class X {class body involving type T}

can have partial template specialization
template<class T, int size>

template <int N> - N is a compile time integer, not a type

template<class T>
template <typename T>

there is no different, typename was introduced because T might not be a class.

template<typename T>
void f(){
    // T::value_type x; this is an error
    typename T::value_type x;

template<class T>
class Set {
public:
    Set() {};
    virtual void add(const T& val);
    int length();
    bool find(const T &val);
private:
    List<T> items;
};

template<class T>
void Set<T>::add(const T &val){
    if(items.find(val)) return;
    items.put(val);
}
template<class T> int Set<T>::length() {return items.length();}
template<class T> bool Set<T>::find(const T&val){
    return items.find(val);
}

this was parametric polymorphism

Functors

  • a callable entity is a function
  • how to recognize? ()
  • a callable entity supports operator()
  • such objects are called function objects or Functors

There are several callable entities in C++

  • function-like macros
  • c functions
  • member functions static and non static
  • pointers to functions
  • ref to functions
  • functors

every function is also a pointer. function pointer -> address of the first point where the function starts.

  • defining a func pointer - int (*pt2Function) (int, char, char);
  • pt2Function = &my_func; // OR pt2Function = my_func
  • a function actually means its address
  • int res = (*pt2Function)(12, 'a','b')

int (A::*pt2Member)(float, char, char);

  • what can we do? assign an address, compare them, call a function using a function pointer, pass a function pointer as an argument..

  • replace switch/ if statements, repalizing user-defined late binding.

smart functions - functors are functions with a state

  • functors are functions with a state, can encapsulate c/c++ function pointers, uses templates and engages polymorphism.

  • has its own type

  • any class that overloads the function call operator

    • void operator() ();
    • int operator()(int, int);
    • double operator()(int, double)
int adder(int a, int b){
    return a + b;
}
class AdderFunctor{
public:
// encapsulated, now it is behaviour of the object
    int operator()(int a, int b){
        return a+b;
    }
};
void main(){
    int x = 5;
    int y = 7;
    int z = adder(x, y);
    AdderFunctor aF;
    int w = aF(x, y); // aF.operator()(x,y)
}
#include <cmath>
#include <functional>

struct less_mag : public std::binary_function<double, double, bool>
{
    bool operator()(double x, double y) const
    {
        return std::fabs(x) < std::fabs(y);
    }
};

  • can pass into sort. less_mag is a function that inherits the overloaded operator.

  • binary_function deprecated
