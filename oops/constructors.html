Constructors & Information Hiding

2 min read · source: oops_cpp/self_notes/7.md
  • Information Hiding
  • Constructor - compiler itself will call, intialization becomes part of the instantiation.
  • Constructor is a member function with thsi pointer
  • Had no return type, not even void
  • why? constructor is instantiating your object, you already have that object, that compiler has decided on the address at which your object is, you have implicit this pointer, no need of return type.

Destructor automatically called when object goes out of scope. No need to remember to allocate and deallocate once you have written constructor and destructor. This is called Resource Acquisition Is Initialization (RAII). It is a programming idiom used in C++ to manage resources such as memory, file handles, and network connections. The idea is that resources are acquired during the initialization of an object and released when the object goes out of scope. This helps to ensure that resources are properly managed and prevents memory leaks and other resource-related issues.

For the destructor there is no descrimination it can semantically do that one object is to be destructed in one way and the other in another way.

Compiler supplies a free default destructor.
Destructor cannot be overloaded

Compiler generated destructor is usually inline, public, non-virtual
Modern C++ preferes unique_ptr because implicit destructor actually calls delete automatically compared to having a raw pointer in your class which won't be deallocated automatically.

Always mention ~A() = default; even if you want to just use the default destructor. Helps when inheritance, long code, template metaprogramming etc.

Lifetime : opening scope of constructor to closing scope of the destructor.

return executs a stack frame, stack frame deallocates.

Execution:

  • memory allocation and binding (memory value with this pointer)
  • constructor call and execution
  • object use
  • destructor call and execution
  • memory deallocation and debinding (memory value with this pointer)

types of object:

  • automatic - allocation in stack, memory allocation and deallocation is implicit by the compiler

  • for array of objects, we MUST provide a default constructor. Complex c[3], even though there is a parameterized constructor, we must provide a default constructor.

  • function don't know how many times it can be called

  • static objects are known before you start executing the code, so there is a static segment

  • when will it get constructed and destructed? since it is in global scope, we don't know. Construction of call static objects before main. So all construction of static before main is called. static has a fixed area and we know layout. Order same as written in file. When will they be destroyed? can be used by any function at any time. Destruction of static objects right after main returns.
