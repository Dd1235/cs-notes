Unique Indexes

2 min read · source: dbms/unique_index.md

A unique index is an index that enforces uniqueness of indexed values.

Example:

CREATE UNIQUE INDEX idx_email
ON users(email);

Now:

INSERT INTO users(email)
VALUES ('a@gmail.com');

works.

But:

INSERT INTO users(email)
VALUES ('a@gmail.com');

fails because the index already contains that key.

How is it implemented?

Suppose the database uses a B+ tree.

Normal index:

alice@gmail.com -> row 1
alice@gmail.com -> row 42
bob@gmail.com   -> row 7

Duplicates are allowed.

Unique index:

alice@gmail.com -> row 1
bob@gmail.com   -> row 7

Before inserting:

charlie@gmail.com

the database traverses the B+ tree.

If key exists:

ERROR: duplicate key

Otherwise:

insert key

PRIMARY KEY vs UNIQUE

Interview favorite.

PRIMARY KEY

CREATE TABLE users(
    id INT PRIMARY KEY
);

Guarantees:

  • Unique
  • Not NULL

Only one primary key per table.

UNIQUE

CREATE TABLE users(
    email VARCHAR(255) UNIQUE
);

Guarantees:

  • Unique
  • NULL handling depends on DBMS

Can have many unique constraints.

Composite Unique Index

Very common.

Example:

CREATE UNIQUE INDEX idx_student_course
ON enrollments(student_id, course_id);

Prevents:

student 1 -> course 5
student 1 -> course 5

from being inserted twice.

But allows:

student 1 -> course 6
student 2 -> course 5

because uniqueness is on the pair.

Why not just use UNIQUE constraint?

These are often effectively the same thing.

email VARCHAR(255) UNIQUE

typically causes the database to automatically create a unique index underneath.

For example, in:

  • PostgreSQL
  • MySQL

a UNIQUE constraint is generally enforced via a unique index.

Multiple NULLs?

Interview trick question.

In many databases:

CREATE TABLE users(
    email VARCHAR(255) UNIQUE
);

allows:

NULL
NULL
NULL

because NULL != NULL in SQL semantics.

Examples include:

  • PostgreSQL
  • MySQL (modern versions)

But behavior can vary across DBMSs.

Performance Impact

Without index:

Check duplicate
→ scan entire table
→ O(n)

With unique B+ tree index:

Lookup key
→ O(log n)

Then insert into index.

Common Interview Questions

Can a unique index speed up queries?

Yes.

It's still an index.

SELECT *
FROM users
WHERE email = 'a@gmail.com';

uses the index for lookup.

Can a table have multiple unique indexes?

Yes.

UNIQUE(email)
UNIQUE(phone)
UNIQUE(username)

all valid.

Is every primary key backed by a unique index?

In most major relational databases, yes.

The database automatically creates an index to enforce the primary key constraint.

A concise interview answer:

A unique index is an index that both accelerates lookups and enforces uniqueness of indexed values. Internally, databases usually check the index before insertion and reject duplicate keys. Primary keys are typically implemented using unique indexes, but a table can have multiple unique indexes while only one primary key.
