Window Functions

2 min read · source: dbms/window_functions.md

GROUP BY collapses rows; a window function computes over a group while keeping every row. That single sentence separates candidates in SQL rounds.

The shape

function() OVER (PARTITION BY … ORDER BY … [frame])
  • PARTITION BY — restart the calculation per group (per user, per month).
  • ORDER BY — gives meaning to rank, lag, and running totals.
  • Frame (ROWS BETWEEN …) — which neighbours feed the calculation; default with ORDER BY is "start through current row" (= running).

Analogy: GROUP BY files students into class-average rows; a window function writes the class average on each student's report card — detail retained, context added.

The worked trio (memorize these shapes)

1 · Top-3 spenders per month — rank inside groups, then filter (window results can't go in WHERE — wrap in a subquery/CTE):

SELECT * FROM (
  SELECT user_id, month, spend,
         DENSE_RANK() OVER (PARTITION BY month ORDER BY spend DESC) AS rnk
  FROM monthly_spend
) t WHERE rnk <= 3;

2 · Running total per account:

SELECT ts, amount,
       SUM(amount) OVER (PARTITION BY account_id ORDER BY ts) AS balance
FROM transactions;

3 · Day-over-day change (LAG):

SELECT day, revenue,
       revenue - LAG(revenue) OVER (ORDER BY day) AS delta
FROM daily_revenue;

The rank trio, disambiguated

On values 100, 90, 90, 80: ROW_NUMBER → 1,2,3,4 (arbitrary tiebreak) · RANK → 1,2,2,4 (gaps) · DENSE_RANK → 1,2,2,3 (no gaps). One line each; interviewers accept nothing fuzzier.

Also in the family: NTILE(4) (quartiles), FIRST_VALUE/LAST_VALUE (beware the default frame cutting off LAST_VALUE — say ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING).

De-dup, the production idiom

Latest row per key: ROW_NUMBER() OVER (PARTITION BY key ORDER BY updated_at DESC) → keep rn = 1. This one query is half of all data-engineering screens.

Drill

  1. Second-highest salary per department — with DENSE_RANK, and why is it cleaner than the self-join version?
  2. 7-day moving average of revenue — write the frame clause.
  3. Users' first order value and its gap to their average order — one query, two window functions.

The Intuit SQL round staple; also Stripe data questions.
