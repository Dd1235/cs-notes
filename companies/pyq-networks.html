PYQ — Networks

2 min read · source: company_wise/pyq_networks.md

Arista's CN screen in question form. Each answer here is the summary; the deep pages carry the full mechanics.

What is DNS, and how does a full lookup flow work — local, root, TLD, authoritative? Arista

DNS maps names → IPs (and more — MX, TXT, CNAME records). The flow, in the order it actually short-circuits: browser cache → OS cache/hosts file → your configured recursive resolver (router/ISP/1.1.1.1). On a full miss, the recursive resolver walks the hierarchy iteratively: a root server (13 anycast identities) refers it to the TLD servers for .com; the TLD refers to the domain's authoritative servers (per its NS records + glue); the authoritative server answers with the A/AAAA record. Every layer caches per the record's TTL, which is why the 3-hop walk almost never happens. Depth for follow-ups — record types, negative caching, anycast, DoH: DNS deep dive.

What happens, networking-wise, when you type a URL into the browser? Arista

The full narrated chain is this site's opening page — what happens when you type a URL — with the flow diagram at the end. The skeleton to recite: parse scheme (HSTS may force https) → DNS resolution (above) → ARP for the gateway's MAC (destination IP isn't in your subnet) → NAT at the router rewrites private source IP/port → TCP three-way handshake to port 443 (TCP deep dive) → TLS 1.3 handshake, ALPN negotiating HTTP/2 or 3 (TLS page) → request → response → render. The differentiating detail for Arista: at every hop the destination IP stays constant while the destination MAC changes — IP routes between networks, MAC delivers on the local segment (L2 switching).

Roles and workings of DHCP, IP, and TCP. Arista

DHCP — how a machine joins a network with zero config. The DORA exchange, over UDP broadcast (ports 67/68): Discover (client broadcasts "any DHCP servers?"), Offer (server proposes an IP + lease), Request (client asks for it — broadcast again, so competing servers know who won), Ack (server confirms: your IP, subnet mask, gateway, DNS servers, lease time). Leases renew at half-life; the fallback 169.254.x.x self-assigned address is the "DHCP failed" tell (addressing notes).

IP — best-effort, connectionless routing between networks: every packet independently forwarded by longest-prefix match on destination address, TTL decremented per hop (traceroute exploits the resulting ICMP errors). No delivery, ordering, or integrity promises — that's deliberate; smartness lives at the edges (routing page).

TCP — the reliability layer on top: connection state via the three-way handshake, a byte stream (no message boundaries — framing is your job), ordering via sequence numbers, reliability via ACKs + retransmission, flow control protecting the receiver (rwnd) and congestion control protecting the network (cwnd). The complete internals — handshake, sliding window, slow start, TIME_WAIT — are the TCP deep dive and flow vs congestion pages.
