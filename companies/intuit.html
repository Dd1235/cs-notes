Company Prep

Intuit

2 min read · source: company_prep/intuit.md

TurboTax/QuickBooks — product engineering at fintech scale. The decider is the Craft Demonstration: build features live in a repo you got 1–2 days earlier, tests included.

Pipeline

  1. Recruiter screen (30–40 min) — background + why Intuit; mission alignment is genuinely scored.
  2. Technical screen (60–90 min, often on Glider) — DSA medium + light system design; recent loops add "how do you validate AI-generated code" style questions.
  3. Craft Demo (~90 min, 2–4 interviewers) — GitHub repo shared 24–48 h ahead; implement user stories live. Scored on: working increments, unit tests, clean naming, and narrating trade-offs.
  4. Onsite loop (4–6 interviews) — coding, design, past-project deep-dive, values/behavioral.

What they actually test

  • Engineering craft over algorithm sport: small correct commits, tests first or alongside, refactor-as-you-go.
  • OOP design conversations — interfaces, composition, SOLID applied to the feature you're building.
  • SQL/DBMS for data-touching teams — joins, indexes, transactions in practical phrasing.
  • Behavioral depth on ownership and customer empathy — have two STAR stories ready.

Revise on this site

Design patterns notes · Testing, SDLC & SE · SQL essentials · Joins · Window functions · ACID · The N+1 problem

Practice prompts (representative, paraphrased)

  1. Clone any small open-source repo; give yourself 60 min to add a feature with tests, narrating aloud. That is the craft demo.
  2. Design a RecurringPayment module — walk the interfaces, failure modes, and how you'd unit-test date logic.
  3. Users table + transactions table: write the query for "top 3 spenders per month" (window functions).
  4. An AI assistant wrote this function for you — how do you decide whether to trust it? (They ask this now.)
