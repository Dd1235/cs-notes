Arista Networks

2 min read · source: company_prep/arista.md

They build switch/router OSes (EOS) in C/C++ and Go. The interview is a systems-programmer filter: networking fluency + raw C/C++ under primitive tooling.

Pipeline

  1. OA (HackerRank, 60–120 min) — coding (lists, trees, DP reported through 2025) plus MCQs on CN, OS, DBMS, C/C++ — the core-subject MCQs are where unprepared candidates die.
  2. Technical rounds (2–3) — coding in deliberately barebones environments: CoderPad, sometimes ssh + a bare editor. Expect to write, compile, and debug without an IDE.
  3. Final/HM round — deeper systems + networking discussion, project walk-through.

What they actually test

  • Networking as a first-class subject: TCP internals (handshake, retransmission, flow vs congestion), sockets, what happens at L2/L3 per hop, subnetting.
  • C/C++ under the hood: pointers, memory layout, object lifetime, virtual dispatch, "what does this print" traps.
  • Debugging as a skill of its own — walking through broken code calmly matters as much as solving fresh problems.
  • Linked-list and tree problems with constraints (in-place, O(1) space) fit the embedded/systems flavour.

Revise on this site

TCP vs UDP deep dive · Sockets & zero-copy · IP & subnetting · L2 switching · I/O multiplexing · Memory layout & padding · vtables · References & const

Practice prompts (representative, paraphrased)

  1. Reverse a linked list in groups of k, then explain where every pointer lives in memory.
  2. Given struct { char a; int b; char c; } — sizeof? Reorder to shrink it.
  3. Walk a TCP segment from write() on host A to read() on host B — every queue and copy.
  4. An interface reports growing retransmissions — list your hypotheses in order and how you'd confirm each.
